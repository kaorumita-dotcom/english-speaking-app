This is likely because Expo Go is loading the app through a web browser (Chrome custom tab)
rather than through the native Expo Go runtime.

Solution: The recording must work in BOTH web and native contexts.
For web: Use navigator.mediaDevices.getUserMedia with proper error handling
For native: Use expo-audio native APIs

Need to detect platform and handle accordingly, with proper fallback for web environments
where mediaDevices may not be available (like Expo Go's embedded browser).
